trueWords API

HTTP API и Python SDK для интеграции детектора русскоязычных текстов, сгенерированных искусственным интеллектом, во внешние приложения и сервисы.

Текущий публичный метод
DependencyAI

На текущем этапе внешний API предоставляет прогноз и расширенный анализ для метода DependencyAI.

Формат доступа
REST + Python SDK

Поддерживаются прямые HTTP-запросы и лёгкий клиентский пакет truewords.

Аутентификация
X-API-Key

Ключ передаётся в HTTP-заголовке и используется для всех защищённых маршрутов API.

Быстрый старт

Ниже приведён минимальный сценарий использования API: сначала проверка готовности сервиса, затем запрос прогноза для метода dependencyai. 

curl https://truewords-api-app.amvera.io/ready

curl -X POST "https://truewords-api-app.amvera.io/v1/predict/dependencyai" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "text": "Это тестовый русский текст для проверки API."
  }'

Аутентификация

Для доступа к защищённым маршрутам используется API-ключ. Ключ передаётся в заголовке X-API-Key.

X-API-Key: YOUR_API_KEY

Базовый адрес API

При использовании облачного развёртывания вызовы отправляются на базовый адрес:

https://truewords-api-app.amvera.io

Для локального запуска можно использовать, например:

http://127.0.0.1:8000

GET /health

Проверка того, что сервер запущен и отвечает на запросы.

Назначение
Проверка доступности сервиса
Аутентификация
Не требуется
curl https://truewords-api-app.amvera.io/health

GET /ready

Проверка готовности методов к работе. Этот маршрут полезен для диагностики развёртывания, загрузки моделей и проверки, какие методы доступны в текущем окружении. 

curl https://truewords-api-app.amvera.io/ready
Пример ответа
{
  "status": "ready",
  "enabled_methods": ["dependencyai"],
  "ready_methods": ["dependencyai"],
  "failed_methods": {}
}

GET /v1/methods

Возвращает сведения о поддерживаемых, включённых и готовых к работе методах. Это полезный маршрут для клиентских приложений, которые должны понимать состав доступных детекторов. 

curl https://truewords-api-app.amvera.io/v1/methods

POST /v1/predict/dependencyai

Быстрый прогноз для метода DependencyAI. Возвращает итоговый вердикт, вероятность класса AI и уверенность в текущем решении.

Тело запроса

{
  "text": "Русскоязычный текст для анализа"
}

Пример запроса

curl -X POST "https://truewords-api-app.amvera.io/v1/predict/dependencyai" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "text": "Русскоязычный текст для анализа"
  }'
Пример ответа
{
  "method": "dependencyai",
  "label": "Human",
  "ai_probability": 0.0870,
  "confidence": 0.9130
}

POST /v1/analyze/dependencyai

Расширенный анализ для метода DependencyAI. Помимо итогового вердикта, этот маршрут возвращает укрупнённую интерпретацию синтаксических сигналов, примеры предложений, данные о надёжности интерпретации и диаграмму в виде base64 PNG. 

curl -X POST "https://truewords-api-app.amvera.io/v1/analyze/dependencyai" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "text": "Русскоязычный текст для анализа"
  }'
Ключевые поля ответа
  • method — имя метода анализа.
  • label — итоговый вердикт: AI или Human.
  • confidence — уверенность в текущем вердикте.
  • summary_text — итоговое текстовое объяснение результата.
  • main_reasons — главные группы синтаксических сигналов.
  • counter_signal — фактор, который частично противоречил итоговому выводу.
  • reliability — оценка устойчивости синтаксической интерпретации.
  • sentence_examples — предложения, наиболее показательные для текущего вердикта.
  • chart_base64_png — диаграмма вклада групп признаков в виде base64 PNG.

Формат ответов

Для пользовательских сценариев обычно достаточно двух полей: label и confidence. Поле ai_probability сохраняется в ответе как техническое, исследовательское поле и может использоваться в сценариях отладки, валидации и ансамблирования.

Интерпретация полей
  • label — итоговый вердикт модели.
  • confidence — уверенность модели в этом вердикте.
  • ai_probability — вероятность класса AI, сохраняемая в API для технических задач.

Ошибки

HTTP-статус Смысл
401 Ошибка аутентификации: неверный или отсутствующий API-ключ.
404 Метод не поддерживается или маршрут не найден.
422 Некорректное тело запроса, чаще всего ошибка JSON или пустой текст.
503 Метод включён, но не готов к работе: не загрузились артефакты или модель.
500 Внутренняя ошибка сервера.

Установка Python SDK

Для Python-приложений доступен клиентский пакет truewords. На текущем этапе его удобно устанавливать прямо из GitHub-репозитория. 

pip install git+https://github.com/SadikovaMaria/truewords-python-sdk.git

Использование SDK

from truewords import TrueWords

client = TrueWords(
    api_key="YOUR_API_KEY",
    method="dependencyai",
    base_url="https://truewords-api-app.amvera.io"
)

result = client.predict("Ваш текст...")
print(result.label)
print(result.confidence)

analysis = client.analyze("Ваш текст...")
print(analysis.label)
print(analysis.confidence)
print(analysis.summary_text)

FAQ

Можно ли использовать только SDK, без прямых HTTP-запросов?

Да. Python SDK предназначен именно для того, чтобы скрыть низкоуровневую работу с HTTP и возвращать удобные Python-объекты.

Какие методы будут доступны дальше?

Архитектура сервиса уже рассчитана на расширение. На следующих этапах в API могут быть добавлены другие детекторы и итоговый ансамблевый режим.
